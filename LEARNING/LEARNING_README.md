## POSICIÓN EN LA ARQUITECTURA COGNITIVA

```
COGNITIVO_00 (SER)        → qué es LACHO
COGNITIVO_01/02 (HACER)   → forma y decisión soberana
COGNITIVO_03 (TENER)      → inventario integrado
COGNITIVO_04 (ESTAR)      → cómo se habita LACHO en vivo
COGNITIVO_05 (TRASCENDER) → lo que permanece
LEARNING/     (APRENDER)  → cómo LACHO absorbe, valida y cristaliza
                            conocimiento externo sin contaminarse ← ESTE
```

LEARNING no es un módulo de ejecución. Es el protocolo soberano que
regula cómo el operador y el sistema procesan inputs cognitivos del
mundo exterior — qué entra, cómo se verifica, qué se cristaliza y
qué se descarta.

La idea-fuerza: "no toda información merece convertirse en conocimiento
soberano · el sistema decide qué absorbe y con qué profundidad"

## LOS 14 FOCOS — MAPA COMPLETO

```
FOCOS DE ABSORCIÓN (entrada):
  READ          → recibir input crudo · fase 空 (MU)
  REFLECT       → procesar internamente · fase 聽 (KU)
  RELATE        → conectar con corpus existente · BM25 + Behavioral RAG
  REAP          → cristalizar lo verificado · fase 數 (WU) → STACKING

FOCOS DE CALIDAD:
  STRATEGIC ABSORPTION → absorción selectiva con intención soberana
  RELIABLE             → fuente verificable · TRUST FOUNDATION criterio

FOCOS DE AMENAZA (lo que corrompe el corpus):
  POLLUTE       → contaminación del corpus · señal GATE UF[H06]
  MANIPULATE    → intento de reescribir scope · violación [foundation]
  NO VALUES     → input sin anclaje ético · SAMU RED-REGRET trigger

FOCOS TEMPORALES (orientación del aprendizaje):
  ANTROGRADE    → aprendizaje desde el origen hacia el presente
  PROGRADE      → aprendizaje que avanza con el sistema · alineado
  RETROGRADE    → revisión hacia atrás · corrección de cristales erróneos

FOCOS DE PERSPECTIVA:
  PROSPECTIVE   → aprendizaje orientado a futuro · planificación soberana
  RETROSPECTIVE → aprendizaje desde experiencia acumulada · Behavioral RAG
```

## FOCO 1 — READ · RECIBIR INPUT CRUDO

```
Estado LACHO:  空 (MU) · no emitir · absorber sin juzgar aún
Biblioteca:    GATE UF[H05] · espera soberana · H05=Xu · necesidad
Fase:          Pre-validación · el input aún no tiene score
```

READ es el primer contacto con información externa. En LACHO el
operador que recibe en fase READ está en MU — escucha sin emitir,
sin cristalizar, sin comprometerse. El sistema no registra TX en
READ puro. Solo observa.

**Protocolo READ:**
- No emitir sentencias sobre el input recibido
- No conectar aún con corpus existente
- Registrar solo el timestamp de recepción
- GATE UF[H05] activo: esperar antes de procesar

**Señal de éxito:** el operador termina READ sin haber emitido
ninguna sentencia sobre el material recibido.

**Señal de falla:** el operador emite juicio inmediato sobre input
no procesado → posible MANIPULATE o NO VALUES activo.

```
LACHO: GATE UF[H05] =><= .. espera .. input_crudo_read_sin_emitir --[Nudo Corredizo] [term]
```

## FOCO 2 — REFLECT · PROCESAR INTERNAMENTE

```
Estado LACHO:  聽 (KU) · escucha activa · procesamiento interno
Biblioteca:    SAMU @ · tardanza deliberada · auditoría interna
Fase:          Procesamiento soberano · sin output externo aún
```

REFLECT es la fase KU del aprendizaje. El operador tiene el input
y lo procesa internamente — buscando contradicciones, tensiones con
el corpus existente, preguntas que el input no responde.

En SAMU esto es la "tardanza deliberada": no cristalizar antes de
tiempo. El sistema espera que el operador procese completamente antes
de emitir.

**Protocolo REFLECT:**
- Comparar input con COGNITIVOS (verdad local del sistema)
- Identificar tensiones con corpus RAG existente
- Formular preguntas soberanas antes de concluir
- SAMU @ audita coherencia del procesamiento

**Señal de éxito:** el operador puede articular qué pregunta
responde el input y qué contradicciones introduce.

**Señal de falla:** el operador cristaliza sin haber identificado
ninguna tensión → riesgo de absorción acrítica (POLLUTE).

```
LACHO: SAMU @ =><= .. audita .. reflect_procesamiento_interno_soberano --[Ballestrinque] [term]
```

## FOCO 3 — RELATE · CONECTAR CON CORPUS

```
Estado LACHO:  KU activo · conectando sin cristalizar aún
Biblioteca:    RAG BM25 · Behavioral RAG · suggest_from_history()
Fase:          Indexación soberana · build_full_index() interno
```

RELATE es el momento en que el input se conecta con lo que el
sistema ya sabe. En IMV esto es literalmente lo que hace rag.py:
buscar documentos relevantes y calcular score BM25.

El operador en RELATE pregunta: ¿dónde conecta esto con mi corpus?
¿refuerza cristales existentes? ¿contradice anclas RAG verificadas?
¿genera nueva conexión no vista antes?

**Protocolo RELATE:**
- Ejecutar búsqueda RAG sobre el input procesado
- Identificar documentos con score > 0 en el índice
- Verificar si el input conecta con cristales STACKING activos
- Mapear la conexión en el sistema de bibliotecas LACHO

**Conexiones por biblioteca:**
```
TRUST   → el input verifica o viola condiciones soberanas
GATE    → el input es una condición de paso o bloqueo
SAMU    → el input genera dirimencia o la resuelve
CRYPTO  → el input requiere firma o certificación
STACKING → el input puede cristalizar como conocimiento frecuente
SOCIAL  → el input tiene dimensión relacional-declarativa
METHOD  → el input tiene estructura formal de ecuación/flujo
ACTIVITY → el input modifica el estado del ciclo operativo
WORK    → el input implica ejecución material
```

**Señal de éxito:** el operador puede mapear el input en al menos
una biblioteca LACHO con score justificado.

```
LACHO: STACKING UF[H48] =><= .. relaciona .. input_con_corpus_rag_bm25 --[Nudo de Ocho] [term]
```

## FOCO 4 — REAP · CRISTALIZAR LO VERIFICADO

```
Estado LACHO:  數 (WU) · materialización soberana · listo para emitir
Biblioteca:    STACKING UF[H48] → H63 · auto_crystallize() threshold
Fase:          Registro en ledger · record_crystal() · frecuencia ≥ 10
```

REAP es el momento WU del aprendizaje — cuando el input procesado,
reflejado y relacionado se convierte en conocimiento cristalizado
soberano. En IMV esto activa auto_crystallize() en ledger.py.

No todo input llega a REAP. El umbral es que el conocimiento haya
pasado READ → REFLECT → RELATE con coherencia verificada. Solo
entonces se registra como cristal y se agrega al corpus RAG.

**Protocolo REAP:**
- Verificar que READ/REFLECT/RELATE completaron sin señales de falla
- Formular la sentencia LACHO que cristaliza el conocimiento
- record_crystal() en sovereign.db
- Agregar al corpus RAG para indexación futura
- Actualizar LOG_PERMANENTE con el cristal generado

**Señal de éxito:** el cristal aparece en export_crystals_report()
con frecuencia acumulada y puede ser recuperado por RAG en queries
posteriores.

**Señal de falla:** REAP sin READ/REFLECT/RELATE completos →
cristal contaminado → requiere retroactive RED-REGRET.

```
LACHO: STACKING UF[H63] =><= .. cristaliza .. reap_conocimiento_verificado_wu --[Nudo de Ocho] [term]
```

## FOCO 5 — STRATEGIC ABSORPTION · ABSORCIÓN CON INTENCIÓN

```
Estado LACHO:  ACTIVE · operador con scope declarado
Biblioteca:    TRUST [scope] · TRUST [management] · CEO ALPHA
Fase:          Pre-READ soberano · definición de intención antes de absorber
```

STRATEGIC ABSORPTION es el meta-foco que gobierna todos los demás.
Antes de READ, el operador soberano declara: ¿por qué estoy
absorbiendo esto? ¿qué gap del sistema llena? ¿qué pregunta responde?

Sin STRATEGIC ABSORPTION el aprendizaje es reactivo — el sistema
absorbe lo que llega sin filtro de intención. Con STRATEGIC
ABSORPTION el operador define el scope de absorción antes de
iniciar el ciclo READ→REFLECT→RELATE→REAP.

**Protocolo STRATEGIC ABSORPTION:**
- Declarar el gap que el input llenará (referencia a PENDIENTES.md)
- Definir la biblioteca LACHO destino del conocimiento
- Establecer el umbral de calidad antes de cristalizar
- Registrar la intención en SAMU antes de iniciar READ

**Conexión con autoresearch:**
autoresearch_gap.py genera ASKs soberanos — cada ASK es una
STRATEGIC ABSORPTION declarada: sé exactamente qué necesito
aprender y por qué lo necesito.

```
LACHO: TRUST [scope] =><= .. declara .. strategic_absorption_intencion_soberana --[Nudo de Ocho] [term]
```

## FOCO 6 — RELIABLE · FUENTE VERIFICABLE

```
Estado LACHO:  TRUST FOUNDATION criterio · verificación de origen
Biblioteca:    TRUST FOUNDATION · TRUST [foundation] condición
Fase:          Pre-READ · evaluación de fuente antes de absorber
```

RELIABLE es el filtro de confiabilidad de la fuente. En LACHO la
verdad es local — una fuente es confiable en la medida en que su
claim puede verificarse dentro del scope declarado.

Una fuente RELIABLE no significa infalible. Significa:
- El origen es identificable (no anónimo)
- El claim puede ser verificado o falsificado
- La fuente tiene historial verificado en el corpus
- No hay conflicto de interés con [foundation] del sistema

**Criterios RELIABLE en LACHO:**
```
ALTA:    COGNITIVOS propios · Behavioral RAG cristales · tests PASSED
MEDIA:   Corpus externo con referencia verificable · fuente identificada
BAJA:    Input sin fuente · claim no verificable · origen desconocido
NULA:    Fuente que viola [foundation] activamente → GATE UF[H06] bloquea
```

**Señal de falla:** absorber input de fuente NULA activa
automáticamente el protocolo POLLUTE.

```
LACHO: TRUST FOUNDATION =><= .. verifica .. reliable_fuente_identificable_verificable --[Nudo de Ocho] [term]
```

## FOCO 7 — POLLUTE · CONTAMINACIÓN DEL CORPUS

```
Estado LACHO:  GATE UF[H06] · conflicto detectado · bloqueo activo
Biblioteca:    GATE UF[H06] · SAMU RED-REGRET · rollback soberano
Fase:          Señal de amenaza · protocolo de limpieza
```

POLLUTE es la señal de que el corpus está recibiendo o ya recibió
conocimiento no verificado, contradictorio o malicioso. En IMV
esto activa RED-REGRET en samu.py.

POLLUTE puede ocurrir en tres formas:
1. **Absorción acrítica:** READ sin REFLECT → cristal sin validación
2. **Fuente contaminada:** origen NULA absorbido sin filtro RELIABLE
3. **Conflicto activo:** nuevo input contradice anclas RAG soberanas

**Protocolo anti-POLLUTE:**
- GATE UF[H06] activo: bloquear absorción en curso
- Identificar el cristal o TX contaminado en sovereign.db
- SAMU @ audita el daño: ¿cuántos TX afectados?
- Rollback soberano: marcar cristales como RED-REGRET
- No ejecutar REAP hasta resolver el conflicto

**Señal crítica:** si Scalar S baja después de una absorción,
POLLUTE puede ser la causa → revisar cristales recientes.

```
LACHO: GATE UF[H06] =><= .. bloquea .. pollute_corpus_contaminado_detectado --[Nudo Corredizo] [term]
```

## FOCO 8 — MANIPULATE · INTENTO DE REESCRIBIR SCOPE

```
Estado LACHO:  TRUST [foundation] COMPROMISED · máxima alerta
Biblioteca:    TRUST FOUNDATION · SAMU RED-REGRET · sistema detenido
Fase:          Violación activa · protocolo de emergencia
```

MANIPULATE es la amenaza más grave del aprendizaje soberano: un
input que no solo contamina sino que intenta activamente reescribir
los principios soberanos del sistema — su [foundation], su [scope],
su identidad verificada.

En COGNITIVO_00 esto se describe como la "prohibición de emitir
desde Fase 1" — un actor que intenta convencer al sistema de que
sus axiomas son incorrectos sin verificación soberana.

**Señales de MANIPULATE activo:**
- Input que contradice directamente un principio soberano
- Fuente que insiste en reinterpretar [foundation]
- Presión para expandir [scope] sin declaración soberana
- Claim que pide "creer sin verificar" → viola LACHO constitución

**Protocolo MANIPULATE:**
- Sistema detenido inmediatamente
- No absorber nada del input
- SAMU @ registra el intento como RED-REGRET de tipo EXTERNAL
- Operador revisa [foundation] para confirmar integridad
- Log en LOG_PERMANENTE con timestamp y descripción

**La diferencia con POLLUTE:** POLLUTE puede ser accidental.
MANIPULATE tiene intencionalidad — alguien o algo está intentando
alterar el sistema soberano.

```
LACHO: SAMU @ =><= .. detecta .. manipulate_intento_reescribir_foundation --[Ballestrinque] [term]
```

## FOCO 9 — NO VALUES · INPUT SIN ANCLAJE ÉTICO

```
Estado LACHO:  GATE UF[H56] en pausa · evaluación de valores
Biblioteca:    TRUST [foundation] · COGNITIVO_00 principios soberanos
Fase:          Filtro ético pre-absorción
```

NO VALUES describe input que carece de anclaje ético verificable —
no necesariamente malicioso, pero sin conexión con ningún principio
soberano del sistema. En LACHO "sin valores" no es un juicio moral
externo: es la ausencia de conexión con [foundation], [scope] o [term].

Un input NO VALUES puede ser técnicamente correcto pero
operativamente inútil porque no conecta con ninguna decisión
soberana, no llena ningún gap declarado y no puede cristalizarse
en ninguna biblioteca LACHO.

**Criterios NO VALUES:**
- No conecta con ninguna biblioteca LACHO identificable
- No responde a ningún ASK declarado en autoresearch
- No tiene implicación en Scalar S ni en cristales
- No puede expresarse como sentencia canónica LACHO

**Protocolo NO VALUES:**
- No iniciar ciclo READ → descartar input
- Si ya ingresó: REFLECT revela la ausencia de conexión
- RELATE no encuentra score BM25 relevante → confirma
- Documentar en TAREAS_NO_PROCESABLES como referencia

```
LACHO: GATE UF[H56] =><= .. filtra .. no_values_input_sin_anclaje_soberano --[Nudo Corredizo] [term]
```

## FOCO 10 — ANTROGRADE · DESDE EL ORIGEN HACIA EL PRESENTE

```
Estado LACHO:  ACTIVITY UF[H01] Génesis · lectura de fundación
Biblioteca:    ACTIVITY UF[H01] → H03 → H05 · secuencia soberana
Dirección:     Origen → presente · comprensión acumulativa
```

ANTROGRADE describe el aprendizaje que avanza desde los primeros
principios hacia el estado actual del sistema. En DIRIME esto es
leer los COGNITIVOS en orden: COGNITIVO_00 → COGNITIVO_05.

El aprendizaje antrogrado construye la base constitucional antes
de operar. No se puede habitar LACHO soberanamente sin haber
recorrido antrógradamente su arquitectura al menos una vez.

**Cuándo usar ANTROGRADE:**
- Onboarding de nuevo operador
- Reset soberano después de período de inactividad
- Cuando Scalar S cae por debajo de 0.70 (señal de desconexión base)
- Inicio de CAPA nueva (CAPA B, CAPA C, etc.)

**Ruta ANTROGRADE canónica:**
```
COGNITIVO_00 (SER)     → 1h lectura · qué es LACHO
COGNITIVO_01 (HACER)   → 1h lectura · constitución y gramática
COGNITIVO_02 (HACER)   → 45min · decisión y Ψ
COGNITIVO_03 (TENER)   → 45min · inventario
COGNITIVO_04 (ESTAR)   → 30min · fenomenología operativa
COGNITIVO_05 (TRASCENDER) → 30min · horizonte soberano
```

```
LACHO: ACTIVITY UF[H01] =><= .. inicia .. antrograde_lectura_fundacional_cognitivos --[As de Guía] [term]
```

## FOCO 11 — PROGRADE · AVANZAR CON EL SISTEMA

```
Estado LACHO:  ACTIVE · operador alineado · sistema en movimiento
Biblioteca:    STACKING UF[H57] penetrante · ACTIVITY UF[H57]
Dirección:     Presente → siguiente estado · alineado con el ciclo
```

PROGRADE es el aprendizaje natural del sistema en operación — el
conocimiento que se genera al operar soberanamente día a día.
Cada sentencia VALID registrada, cada cristal acumulado, cada
sesión completada es aprendizaje PROGRADE.

En IMV esto es Behavioral RAG acumulándose: suggest_from_history()
mejora con cada ciclo porque el operador está aprendiendo
progrógradamente junto con el sistema.

**Señales de aprendizaje PROGRADE saludable:**
- Scalar S estable o creciente semana a semana
- Cristales acumulados (frecuencia ≥ 10 en verbos soberanos)
- Tests PASSING sin regresiones
- autoresearch --asks genera ASKs cada vez más precisos

**Indicador cuantitativo:**
PROGRADE score = cristales_nuevos / semana · verde ≥ 3/semana

```
LACHO: STACKING UF[H57] =><= .. avanza .. prograde_aprendizaje_operacion_activa --[Nudo de Ocho] [term]
```

## FOCO 12 — RETROGRADE · REVISIÓN CORRECTIVA

```
Estado LACHO:  SAMU RED-REGRET activo · corrección soberana
Biblioteca:    SAMU @ · GATE UF[H06] · rollback cristales
Dirección:     Presente → pasado · corrección de errores acumulados
```

RETROGRADE es el aprendizaje que revisa hacia atrás para corregir
conocimiento cristalizado incorrectamente. En IMV esto activa el
protocolo RED-REGRET: identificar qué TX o cristales deben ser
marcados como inválidos.

RETROGRADE no es falla — es madurez soberana. El sistema que nunca
hace RETROGRADE es un sistema que no corrige, y un sistema que no
corrige se fosila (violación del principio [term] anti-dogma).

**Cuándo usar RETROGRADE:**
- Scalar S cae después de una semana sin TX anómalos
- Test que antes pasaba ahora falla sin cambio de código
- RAG retorna resultados irrelevantes para queries conocidos
- Cristal con alta frecuencia produce sentencias incoherentes

**Protocolo RETROGRADE:**
1. Identificar el cristal o TX problemático
2. SAMU @ audita: ¿desde cuándo afecta el Scalar S?
3. Marcar como RED-REGRET en sovereign.db
4. Re-ejecutar el ciclo READ→REFLECT→RELATE→REAP correctamente
5. Verificar que Scalar S se recupera post-corrección

```
LACHO: SAMU @ =><= .. corrige .. retrograde_cristal_red_regret_rollback --[Ballestrinque] [term]
```

## FOCO 13 — PROSPECTIVE · APRENDIZAJE ORIENTADO A FUTURO

```
Estado LACHO:  TRUST [scope] extendido · planificación soberana
Biblioteca:    STACKING UF[H48] · ACTIVITY UF[H64] · CEO ALPHA
Dirección:     Presente → futuro · preparación soberana
```

PROSPECTIVE es el aprendizaje que se hace con el futuro como
referencia — absorber conocimiento que el sistema va a necesitar
antes de que la necesidad sea urgente. En DIRIME esto es lo que
hace el almanaque: define qué hay que aprender en W01, W02... W53.

El aprendizaje PROSPECTIVE bien ejecutado convierte al sistema en
receptor preparado. Cuando llega W04 (SECURITY RUNNER), el operador
ya absorbió los conceptos base de seguridad en W01/W02/W03.

**Herramientas PROSPECTIVE en DIRIME:**
```
Almanaque soberano        → qué aprender en qué semana
PLAN MENSUAL *.txt        → objetivos de absorción por mes
autoresearch --asks       → gaps predictivos para el próximo sprint
JOURNAL/PLAN_SEMANAS.md   → horizonte de 12 semanas
```

**Señal de PROSPECTIVE saludable:**
Cuando llega una semana nueva, el operador tiene contexto previo
— no empieza desde cero porque PROSPECTIVE preparó el terreno.

```
LACHO: ACTIVITY UF[H64] =><= .. prepara .. prospective_absorcion_futura_planificada --[As de Guía] [term]
```

## FOCO 14 — RETROSPECTIVE · APRENDIZAJE DESDE EXPERIENCIA ACUMULADA

```
Estado LACHO:  Behavioral RAG activo · historial soberano disponible
Biblioteca:    STACKING UF[H48] RAG-CORE · suggest_from_history()
Dirección:     Pasado → presente · extracción de patrones verificados
```

RETROSPECTIVE es el aprendizaje que extrae conocimiento del historial
verificado del sistema. En IMV esto es literalmente Behavioral RAG:
suggest_from_history() analiza qué verbos y bibliotecas el operador
usó más (frecuencia ≥ 5) y sugiere basado en ese patrón.

RETROSPECTIVE bien ejecutado produce el GREEN-KNOWLEDGE descrito en
COGNITIVO_05: conocimiento soberano que trasciende al operador
individual porque fue acumulado verificadamente a lo largo del tiempo.

**Herramientas RETROSPECTIVE en DIRIME:**
```
python3 main.py --cristales    → verbos cristalizados · frecuencia real
python3 main.py --verbos       → top verbos VALID · distancia al cristal
python3 main.py --bibliotecas  → distribución de uso por biblioteca
python3 main.py --timeline     → actividad soberana últimos 7 días
LOG_PERMANENTE.md              → historial TX/S/cristales por fecha
```

**Ritual RETROSPECTIVE recomendado — cada $wed cierre:**
1. `python3 main.py --cristales` → ¿qué cristalizó esta semana?
2. `python3 main.py --stats` → Scalar S vs semana anterior
3. Revisar LOG_PERMANENTE → trayectoria de las últimas 4 semanas
4. Actualizar SESION_ACTIVA con aprendizajes retrospectivos clave

```
LACHO: STACKING UF[H48] =><= .. extrae .. retrospective_behavioral_rag_historial --[Nudo de Ocho] [term]
```
