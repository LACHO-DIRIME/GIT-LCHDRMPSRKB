## RITUAL SEMANAL ESTÁNDAR — 7 DÍAS CANÓNICOS

```
DÍA 1 — $thu (MAÑANA):
  1. Leer entrada de la semana en ALMANAQUE_SOBERANO.md
  2. Declarar STRATEGIC ABSORPTION en SESION_ACTIVA.md
  3. Ejecutar READ del material predefinido
  4. REFLECT: anotar 3 tensiones con corpus existente

DÍA 2 — $thu (TARDE):
  5. RELATE: búsqueda RAG sobre el tema de la semana
  6. Verificar: ¿el input pasa el filtro RELIABLE?
  7. Ejecutar prompts Windsurf del día

DÍA 3 — $thu (NOCHE):
  8. REAP: verificar cristales generados
     cd ~/DIRIME/IMV && python3 main.py --cristales
  9. RETROSPECTIVE: ¿Scalar S mejoró? ¿qué cristalizó?
  10. Actualizar almanaque: marcar semana ✅

DÍA 4 — $thu (VIERNES):
  11. Declarar STRATEGIC ABSORPTION para semana siguiente
  12. Preparar material predefinido si aplica
  13. Actualizar LOG_PERMANENTE con cierre semanal

DÍA 5 — $thu (SÁBADO):
  14. RETROSPECTIVE semanal profundo
  15. Actualizar PLAN_SEMANAS.md con aprendizajes
  16. Actualizar LEARNING/README.md con nuevas lecciones

DÍA 6 — $thu (DOMINGO):
  17. Descanso operador (opcional: lectura técnica)
  18. Preparar material para próxima semana

DÍA 7 — $thu (LUNES):
  19. Cierre semanal completo
  20. Commit soberano semanal
  21. Celebrar logros y preparar siguiente ciclo
```

## RITUAL SEMANAL EXPANDIDO — 14 DÍAS

```
DÍA 1 — $thu (MAÑANA): INICIO + READ
  - Leer almanaque W+1 completo
  - Declarar STRATEGIC ABSORPTION
  - Ejecutar READ del material predefinido (1h)
  - REFLECT: 3 tensiones + conexiones corpus

DÍA 2 — $thu (TARDE): RELATE + PRIMEROS PROMPTS
  - Búsqueda RAG sobre tema semanal (2h)
  - Verificar scores BM25 > 1.0
  - Ejecutar PROMPT_01 Windsurf
  - Validar cambios en código/tests

DÍA 3 — $thu (MIÉRCOLES): REAP + NERVE CELLS
  - Verificar cristales generados (30min)
  - Crear Nerve Cell del día (LACHO VALID)
  - RAG indexación con boost × 2.0
  - Tests de validación del Nerve Cell

DÍA 4 — $thu (JUEVES): PROGRESO CENTRAL
  - Avanzar en tareas principales del sprint
  - Tests parciales de componentes críticos
  - Actualizar métricas en SESION_ACTIVA.md
  - Preparar demostración si aplica

DÍA 5 — $thu (VIERNES): INTEGRACIÓN + TESTING
  - Integrar componentes del día
  - Tests de integración completa
  - Revisión de arquitectura del sprint
  - Documentar cambios para retroalimentación

DÍA 6 — $thu (SÁBADO): REVISIÓN + PREPARACIÓN
  - Revisión semanal completa
  - Identificar patrones de éxito/fracaso
  - Preparar material para próxima semana
  - Actualizar documentación

DÍA 7 — $thu (LUNES): CIERRE SEMANAL
  - Verificar cumplimiento semanal completo
  - Actualizar LOG_PERMANENTE con cierre
  - Actualizar ALMANAQUE_SOBERANO.md (marcar ✅)
  - Commit soberano semanal
  - Declarar STRATEGIC ABSORPTION W+1
  - Celebrar logros del ciclo
```

## ADAPTACIÓN POR TIPO DE SEMANA

### Semanas ESTÁNDAR (7 días) - Para operación normal
```
Foco: READ→REFLECT→RELATE→REAP
Ritual: 7 días canónicos
Intensidad: Moderada
Objetivo: Operación estable del sistema
```

### Semanas INTENSIVAS (14 días) - Para sprints/cambios mayores
```
Foco: PROFUNDIZACIÓN + EXPANSIÓN
Ritual: 14 días con milestones diarios
Intensidad: Alta
Objetivo: Entregar hito del bloque/cambio mayor
```

### Semanas MEGA-INTENSIVAS (21 días) - Para hitos anuales
```
Foco: INVESTIGACIÓN + INNOVACIÓN + CIERRE
Ritual: 21 días con investigación profunda
Intensidad: Máxima
Objetivo: Hitos anuales (V1.0.0, aniversario, etc.)
```

### Semanas DE RECUPERACIÓN (3-5 días) - Post RETROGRADE
```
Foco: RECONSTRUCCIÓN + ESTABILIZACIÓN
Ritual: Corto, enfocado en recuperar estado estable
Intensidad: Baja-Moderada
Objetivo: Volver a operación normal sin degradación
```

## INTEGRACIÓN CON ALMANAQUE SOBERANO

### Flujo semanal completo
```bash
# 1. $thu MAÑANA: Leer almanaque
grep -A 15 "W[0-9][0-9]" ~/DIRIME/JOURNAL/ALMANAQUE_SOBERANO.md

# 2. Durante la semana: Seguir ritual según tipo
#    - ESTÁNDAR: 7 días canónicos
#    - INTENSIVO: 14 días con milestones
#    - MEGA-INTENSIVO: 21 días para hitos
#    - RECUPERACIÓN: 3-5 días post-RETROGRADE

# 3. $thu NOCHE: Actualizar almanaque
#    Marcar semana actual con estado final
#    - Agregar aprendizajes clave a LEARNING/README.md
#    - Actualizar LOG_PERMANENTE con cierre semanal

# 4. $thu NOCHE: Preparar siguiente semana
#    Leer entrada W+1 del almanaque
#    - Verificar prerequisitos
#    - Ajustar intensidad del ritual según tipo de semana
```

### Mapeo de tipo de semana → ritual
```bash
# Determinar tipo de semana basado en almanaque
WEEK_TYPE=$(grep -A 5 "W[0-9][0-9]" ~/DIRIME/JOURNAL/ALMANAQUE_SOBERANO.md | \
  grep -o "ESTÁNDAR\|INTENSIVO\|MEGA-INTENSIVO\|RECUPERACIÓN" | \
  head -1 | awk '{print $2}')

case $WEEK_TYPE in
  "ESTÁNDAR") echo "7_dias_canonico.sh" ;;
  "INTENSIVO") echo "14_dias_intensivo.sh" ;;
  "MEGA-INTENSIVO") echo "21_dias_mega_intensivo.sh" ;;
  "RECUPERACIÓN") echo "recuperacion_post_retrograde.sh" ;;
esac
```

### Scripts de ritual especializado
```bash
# 7_dias_canonico.sh → ejecuta ritual estándar de 7 días
# 14_dias_intensivo.sh → ejecuta ritual intensivo con milestones diarios
# 21_dias_mega_intensivo.sh → ejecuta ritual mega-intensivo para hitos anuales
# recuperacion_post_retrograde.sh → ejecuta ritual de recuperación corta
```
