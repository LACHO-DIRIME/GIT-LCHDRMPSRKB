# ELPULSAR.APP — ¿Qué es exactamente?

## OPCIÓN A: elpulsar.app es la MARCA DEL PRODUCTO FINAL (B2C)
El usuario final ve y usa "elpulsar.app". GUALICHO HUINCA es el nombre
legal/corporativo que el usuario nunca ve.
Analogía: Mercado Pago es la marca, MercadoLibre es la empresa.
Consecuencia: elpulsar.app necesita UX pensada para usuario final.
El sitio web principal ES elpulsar.app.

## OPCIÓN B: elpulsar.app es el DASHBOARD DEL OPERADOR (B2B)
Las empresas que usan la infraestructura KALIL acceden via elpulsar.app
para monitorear sus operaciones, ver Scalar S, ejecutar sentencias LACHO.
El usuario final usa productos con otras marcas (white-label).
Analogía: Stripe Dashboard — el comercio lo usa, el consumidor no lo ve.
Consecuencia: elpulsar.app es una herramienta profesional, no una app de consumo.

## OPCIÓN C: elpulsar.app es el API GATEWAY (infraestructura pura)
elpulsar.app es la URL base de la API que otros construyen sobre ella.
GUALICHO HUINCA es uno de los primeros clientes de su propia infraestructura.
Analogía: Twilio — desarrolladores construyen productos sobre la API.
Consecuencia: el negocio principal es B2B2C. Requiere documentación técnica excelente.

## RECOMENDACIÓN DEL SISTEMA
Dado el estado actual (DIRIME corriendo, LACHO operativo, 4 prácticas
implementadas), la Opción B es la más coherente con lo que ya existe:
elpulsar.app como dashboard soberano donde el operador controla
sus nodos KALIL, visualiza el ledger, ejecuta operaciones.
GUALICHO HUINCA como marca del servicio hacia los clientes finales.
