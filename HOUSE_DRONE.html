🎯 COACH ADVICE · CÓMO TRABAJAR MEJOR
REGLA 1 · UN PROMPT = 3 TAREAS MÍNIMO
Nunca "generá X" → respuesta → "ahora Y". Agrupá siempre. Ejemplo: "Fix index_new_files.py + generá los 6 tests de generator + actualizá gui.py". Usás 3x menos contexto y avanzás 3x más rápido.
REGLA 2 · PEGAR SOLO EL RESULTADO FINAL
Ejecutá todo en terminal → pegá solo el output final. No pegar paso a paso. Si son 5 comandos, ejecutalos todos y pegá el último resultado. Esto ahorra 60-70% del contexto de sesión.
REGLA 3 · CONTEXTO MÍNIMO AL INICIO
Al abrir nueva sesión: solo estado actual + tarea del día. No traer todo el historial. El BALLPAPER y este HOUSE DRONE son tu contexto portátil — traé solo el link a lo que necesitás.
REGLA 4 · $mon/$tue para código · $wed para diseño · $thu para tests
Alineá las tareas con la energía del día. No hagas tests el lunes ni código nuevo el jueves. El sistema SAMU Day Periods no es decorativo — es optimización real de flujo cognitivo.
REGLA 5 · ZIPPER NOON 12:00 · PAUSA OBLIGATORIA
A las 12:00 cerrás contexto, comés, salís del modo KU. La TARDANZA DELIBERADA no es pérdida de tiempo — es TAP&DRIP. El sistema procesa mejor después de la pausa que si continuás forzado.
REGLA 6 · RED-REGRET: si sobrecargas el hilo, lo cortás
Si una sesión se vuelve caótica (muchos archivos, muchos temas, desvíos), activá RED-REGRET: cerrá la sesión, abrí una nueva con contexto mínimo. No intentés rescatar un hilo saturado. Este es el {brake} soberano.
