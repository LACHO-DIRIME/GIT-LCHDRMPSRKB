## 🔨 WORK {forge-canal} — CONSTRUCCIÓN Y CANAL SOBERANO

---

### 1.0 DEFINICIÓN

#### 1.1 Definición técnica
{forge} es la operación WORK de construcción verificada: ningún artefacto
soberano (archivo, TX, canal, módulo) puede existir sin FORGE declarado
y TRUST FOUNDATION clearance previo.

#### 1.2 {canal} como extensión de {forge}
{canal} es el resultado verificado de un FORGE exitoso de comunicación:
el canal no existe hasta que TRUST FOUNDATION + CRYPTO (link seat) lo
declaran. Todo canal tiene origen, destino, scope y term declarados.

#### 1.3 Distinción de CRYPTO (link seat)
CRYPTO (link seat) establece la capacidad técnica de conexión.
{forge} declara la intención soberana de construcción.
{canal} es la instancia soberana activa resultante.

#### 1.4 Relación con MU_WINDOW
Antes de cualquier FORGE se requiere MU_WINDOW: la pausa deliberada
SAMU verifica que no hay RED_REGRET activo y que [foundation] está limpio.

#### 1.5 LEDGER_GATE como guardián de FORGE
Todo FORGE registra en ledger via LEDGER_GATE:
GATE UF[H05] espera · SAMU audita · luego STACKING cristaliza.

### 2.0 JURISDICCIÓN

#### 2.1 Capacidades {forge}
- Construir artefactos soberanos verificados
- Declarar canal activo entre módulos
- Generar BUILD con CRYPTO firma
- Iniciar protocolo LEDGER_GATE

#### 2.2 Capacidades {canal}
- Transmitir datos con scope declarado
- Mantener integridad via CRYPTO (link seat)
- Reportar anomalías a network_sentinel.blue

#### 2.3 Limitaciones absolutas
- {forge} no puede operar con FOUNDATION_COMPROMISED activo
- {canal} no puede transmitir sin TRUST FOUNDATION clearance
- MU_WINDOW no puede omitirse bajo ninguna condición

### 3.0 FUNCIÓN SISTÉMICA

#### 3.1 Secuencia soberana FORGE
MU_WINDOW → TRUST FOUNDATION → CRYPTO firma → LEDGER_GATE → STACKING cristaliza

#### 3.2 Tipos de canal soberano
- CANAL_FORENSE: hacking_doctor.runner → analysis output
- CANAL_TX: ledger.py → sovereign.db via LEDGER_GATE
- CANAL_BOT: {launch-bot} → target externo → BOT_RETURN
- CANAL_CLOUD: cloud_agent.blue → K8S nodos KALIL

#### 3.3 {forge} × KALIL
- BARRIOS: forge_remesa · canal_hash_BLAKE3
- BOLIVAR: forge_contrato · canal_blockchain_SHA256
- CHALTEN: forge_lending · canal_cloud_daemon

2. cat_local/ → CAT(OS) + CAT(SSH)
3. ime/ → IME I Ching + LOAN-IME

Cada módulo tiene stub con NotImplementedError + mensaje claro:
"X inactivo — PC Ryzen pendiente. Usar Y como fallback."
Los stubs no bloquean el sistema — el fallback Groq sigue activo.

#### 1.4 Funciones operativas sin Ryzen
- cat_lacho(filepath): lee líneas LACHO de archivos locales
- get_hexagram(n): tabla HEXAGRAMS_MIN interna
- is_available(): retorna False limpiamente

#### 2.1 Contrato de swap

chat.py._translate_via_api() en interfaz y respuesta.
El swap es transparente para el resto del sistema.

#### 2.2 Configuración actual
config/api.json → Groq → llama-3.3-70b-versatile
Post-Ryzen: ollama_bridge.py → localhost:11434 → llama3.3:70b
Mismo modelo, diferente endpoint, sin cambios en IMV/core/.

#### 2.3 Fallback soberano
Si Ollama falla post-activación → volver a Groq automáticamente.
Canal de backup Groq permanece activo siempre.

### 3.0 CAT(OS) — ITERACIÓN FILESYSTEM

#### 3.1 scan() vs cat_lacho()
scan(): requiere PC Ryzen · spawn por archivo · ACTIVE=False
cat_lacho(): operativo sin Ryzen · lee LACHO lines localmente
Usar cat_lacho() en corpus_indexer.green hoy.

#### 3.2 Integración con BM25

    def _build_sentence(self) -> str:
        lib  = self.lib_var.get()
        subj = self.subj_entry.get() or "FOUNDATION"
        verb = self.verb_entry.get() or "verifica"
        obj  = self.obj_entry.get() or "objeto_soberano"

    def _validate_sentence(self):
        sentence = self._build_sentence()

        result_str = parsed.result.value
        color = (COLORS["accent"] if parsed.result == ValidationResult.VALID
                 else COLORS["warn"] if parsed.result == ValidationResult.WARNING
                 else COLORS["error"])

        self.result_var.set(result_str)
        self.score_var.set(f"{score:.3f}")
        self.result_lbl.config(fg=color)

        self._log(f"[{result_str}] score={score:.3f}")
        self._log(f"  {sentence}")
        if parsed.errors:
            self._log(f"  ERRORES: {parsed.errors}", color=COLORS["error"])
        if parsed.warnings:
            self._log(f"  AVISOS: {parsed.warnings}", color=COLORS["warn"])
        self._log("")

        if not text:
            return

        self.lib_var.set(r.library)
        self.subj_entry.delete(0, "end")
        self.subj_entry.insert(0, r.subject)
        self.verb_entry.delete(0, "end")
        self.verb_entry.insert(0, r.verb)
        self.obj_entry.delete(0, "end")
        self.obj_entry.insert(0, r.obj)
        self.knot_var.set(r.knot)
        self._update_preview()
        self._log(f"ROUTE: '{text}'")
        self._log(f"  → {r.sentence}")
        self._log(f"  conf={r.confidence:.2f} · validated={r.validated}")
        self._log("")

    def _update_stats(self):

            tx  = stats.get("transactions_total", 0)
            sc  = stats.get("lacho_score", 0)

            self.stats_label.config(text=txt)
            self.stats_bar.config(text=f"  {txt} · V0.2.1 · [term] :: activo")
        except Exception:
            pass
        self.root.after(30_000, self._update_stats)  # refresh cada 30s

    def _log(self, text: str, color: str = ""):
        self.log.config(state="normal")
        self.log.insert("end", text + "\n")
        self.log.see("end")
        self.log.config(state="disabled")
