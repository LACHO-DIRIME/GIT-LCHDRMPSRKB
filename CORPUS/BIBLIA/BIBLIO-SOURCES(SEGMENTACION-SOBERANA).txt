### 2.0 TOMO — unidad mínima de coherencia

Definición corpus (COGNITIVO_02):
"Mínimo donde el sistema puede decir: en este scope,
bajo estas condiciones, esto es coherente."

ETIQUETA: alfabética base-26 extendida, referenciada al TERM activo.
  A → Z           (nivel 1 —     26 TOMOs)
  AA → ZZ         (nivel 2 —    676 TOMOs)
  AAA → ZZZ       (nivel 3 — 17.576 TOMOs)
  Techo declarado: ZZZZZZZZ

FORMATO:   [term:TOMO_ID]
EJEMPLOS:  [term:A]  [term:BC]  [term:AAZ]  [term:MXYZ]

RELACIÓN CON CRISTALES IMV:
  Un cristal verificado (verbo >= 10x VALID) es un TOMO mínimo.
  TOMO_ID se asigna en orden de cristalización:
    [term:A]  → materializa  (cristal 1)
    [term:B]  → verifica     (cristal 2)
    [term:C]  → conecta      (cristal 3)
    ...
    [term:AN] → contiene     (cristal 40)

LECTURA SOBERANA:
  [term:A]   → primer TOMO del árbol TERM activo
  [term:AA]  → vigésimo séptimo TOMO — segunda vuelta
  [term:AAA] → TOMO profundo — tercer ciclo anidado

====================================================================
### 3.0 SLICE — corte fino dentro de un TOMO

ETIQUETA: grados enteros 0 a 360 con prefijo °
FORMATO:  °NNN
EJEMPLOS: °0  °45  °90  °180  °270  °360

CONVENCIÓN SOBERANA:
  °0   → apertura — inicio del TOMO
  °90  → desarrollo activo
  °180 → dirimencia — punto medio soberano
  °270 → cristalización
  °360 → cierre soberano — abre el siguiente TOMO

MAPEO A HEXAGRAMAS UF:
  °0   - °45  → UF[H01-H08]   génesis / inicio
  °45  - °90  → UF[H09-H16]   desarrollo
  °90  - °180 → UF[H17-H32]   tensión activa
  °180 - °270 → UF[H33-H48]   transformación
  °270 - °360 → UF[H49-H64]   cierre / cristalización

====================================================================
### 4.0 SEGMENT — partición principal de comprensión

ETIQUETA: número romano con prefijo §
FORMATO:  §I  §II  §III ... §XCIX
EJEMPLOS: §I  §IV  §XII

FUNCIÓN: divide el TOMO en bloques de comprensión distintos.
         Cada segmento tiene coherencia interna verificable.
         Análogo a los §§ de los BIBLIO-SOURCES del corpus.

====================================================================
### 5.0 CLUSTER — agrupación temática

ETIQUETA: nombre soberano en MAYÚSCULAS con prefijo #
FORMATO:  #NOMBRE
EJEMPLOS: #CORE  #MERCADO  #NOTARIA  #CUSTODIA  #CICLO  #FABRIC

CLUSTERS PREDEFINIDOS DEL PROYECTO:
  #CORE       → fundamentos TRUST/FOUNDATION/SCOPE/TERM
  #MERCADO    → MEMECOIN / volatilidad / estrategia económica
  #NOTARIA    → registro soberano / certificación / sellado
  #CUSTODIA   → protección / acceso / identidad soberana
  #CICLO      → temporal / scheduler / ciclo semanal
  #FABRIC     → ledger / HL FABRIC / trazabilidad
  #SEGMENTACION → sistema TOMO/SLICE/SEGMENT/CLUSTER/OFFSET
  #TETHER     → vinculación inter-nodo / anclaje soberano

====================================================================
### 6.0 OFFSET — índice de desplazamiento exacto

ETIQUETA: hexadecimal con prefijo 0x
FORMATO:  0xNN  o  0xNNNN
EJEMPLOS: 0x00  0x1A  0xFF  0x0100

FUNCIÓN: referencia posición exacta en ledger, corpus o transacción.
  0x00 → inicio absoluto del bloque
  0xFF → posición 255 — fin de bloque estándar
  0x0100 → posición 256 — inicio de segundo bloque

USO EN LEDGER: referencia la transacción exacta dentro de un TOMO.
USO EN CORPUS: referencia el byte/línea exacta de un BIBLIO-SOURCES.

### 8.0 TETHER — vínculo soberano entre nodos

Integración de tethering en la arquitectura de segmentación.

8.1 TETHER como extensión de archivo (.tether)
  Archivo de vínculo entre NODO-WS y NODO-IDE.
  Contiene: TOMO_ID origen, TOMO_ID destino, canal CRYPTO activo.
  Ubicación natural: DIRIME_v2/fabric/
  Ejemplo: ws_to_ide_[term:A]_to_[term:BC].tether
  Estado: PENDIENTE — requiere dos nodos físicos (PC nueva)

8.2 TETHER como módulo SOCIAL {tether}
  Módulo de vinculación persistente entre operador y sistema.
  Análogo a {relay} pero con estado mantenido entre sesiones.
  Diferencia con {relay}: {relay} transfiere, {tether} ancla.
  Estado: DOCUMENTADO — implementación en DIRIME_v2/scheduler/

8.3 TETHER como asiento CRYPTO (tether seat)
  Séptimo asiento de autoridad CRYPTO.
  Opera entre (link seat) y (flow seat).
  Mantiene canal verificado durante sesión completa inter-nodo.
  Estado: DOCUMENTADO — implementación en DIRIME_v2/fabric/

8.4 TETHERING como operación soberana
  CRYPTO (tether seat) =><= .. ancla .. canal_inter_nodo --[Nudo de Ocho] [term]
  SOCIAL {tether} =><= .. vincula .. operador_sistema --[As de Guía] [term]
