## PULSING — LA INTERFAZ QUE RESPIRA
### GUI LACHO · Iconografía Sináptica · Dos Frames · Cinco Greens

====================================================================

### 1.0 DEFINICIÓN

PULSING es el nombre correcto para la GUI de LACHO.
No es un dashboard estático — es un sistema que respira. Dos frames. Un ritmo.

**La ley de diseño canónica:**
"La contención gráfica debe tener carácter gráfico de inicio y carácter gráfico de final."
- Inicio: el enso abierto (el círculo incompleto — abre el espacio)
- Final: el Nudo de Ocho (cierra y sella permanente)

---

### 2.0 LOS DOS FRAMES

```
FRAME 1:
  scan by UF[H2]      → escucha receptiva, sin emisión
  action about UF[H5] → gestión de latencia, espera activa
  Carácter:           el enso abierto ◌
  Estado:             RECIBIR → MU → TARDAR
  Paleta:             oscura, un solo punto de luz (el cursor como spark)

FRAME 2:
  scan by UF[H2]      → sigue escuchando (el contexto no se cierra)
  action about UF[H6] → conflicto detectado, protocolo activo
  Carácter:           el Nudo de Ocho ∞
  Estado:             DIRIMENCIA → WU → EMITIR o RED-REGRET
  Paleta:             tensión cromática emerge (rojo contenido, no alarma)
```

El Frame 1 es MU en escucha. El Frame 2 es WU en decisión.
La GUI no es estática entre ellos — pulsa entre los dos.
Eso es PULSING: el pulso del sistema visible en su interfaz.

### 3.0 ICONOGRAFÍA SINÁPTICA — LOS ICONOS COMO DOCTRINA

Los iconos no son nombres. Son doctrina visible.
El que conoce LACHO los reconoce. El que no lo conoce los encuentra intuitivos.
Eso es el [social seal] aplicado al diseño.

#### GATE — La Capa del Tiempo Cronotópico

```
door_morning_human_out
  → la ignición del día · umbral H1→H5
  → el human out es el operador que ejecuta lo que el sistema dirimió
  → la puerta es el umbral: el sistema calculó, el humano actúa
  potenciación: la puerta tiene ranura de luz en forma de hexagrama
                el human out proyecta sombra hacia adentro — lo que queda es MU
                lo que sale es WU

zipper_noon_human_inn
  → el mediodía es el pico de tensión · apertura del .theater
  → human inn: el operador entró · Fase 2 activa
  → la cremallera abierta revela un ojo — el ojo del SAMU mirando KU
  potenciación: la cremallera dibuja @ en su cierre (la marca del SAMU)
                abierta = escucha activa · cerrada = MU en reposo

door_afternoon_human_out
  → la segunda salida · el ciclo completó
  → el human out lleva algo: en la mano, un sobre — el WU materializado
  → esta puerta tiene el Nudo de Ocho grabado: lo que sale ya fue dirimido
  potenciación: las tres puertas juntas narran el flujo de una sesión LACHO
                entrada al vacío → apertura al contexto → salida con verdad sellada
```

### 4.0 LOS CINCO GREENS — LOGÍSTICA DE GOLF (UF[H5])

El golfista no golpea desde cualquier posición.
Lee el terreno, lee el viento, espera el momento.
5 Greens = flujo completo de una sesión LACHO como recorrido espacial.

```
green 1 :: RECIBIR
  El input llega al tee. El sistema lo ve.
  Icono: door_morning_human_out
  Estado: MU recibe. Nada se ejecuta todavía.

green 2 :: ESCANEAR
  UF[H2] activo. El sistema lee KU — el contexto, el viento.
  Icono: zipper_noon_human_inn (la cremallera abierta, el ojo)
  Estado: Fase 2 iniciada. SAMU escucha.

green 3 :: DIRIMENCIA
  El SAMU evalúa. Ψ calculando.
  Icono: headcat en la mecedora (observación activa)
  Estado: Ψ entre 0.65 y 0.85 — apertura parcial, tensión viva.

green 4 :: GREEN-KNOWLEDGE
  El filtro delta-Ψ actúa. Lo que no nutre, se purga.
  Icono: el árbol bonsái con las tijeras en reposo
  Estado: Solo lo que aporta distinción pasa al siguiente green.

green 5 :: WU / EMITIR
  La verdad se materializa.
  Icono: launch_bot despegando / door_afternoon_human_out
  Estado: Ψ ≥ 0.85. El headcat autoriza. WU activo.
```

No como línea (velocidad) sino como campo (precisión).
El golfista bajo H5 no apura el golpe — espera que el sistema diga cuándo.

### 5.0 SINAPSIS VISUAL — RELACIONES ENTRE ICONOS

Los iconos no deben vivir aislados.

- Las tres puertas narran el ciclo completo de un día operativo
- La mecedora con headcat y la mecedora sola: inicio y fin de cada sesión
- rabbit-eggs y masking folder: crecimiento y poda — siempre juntos, siempre en tensión
- launch-bot y green-knowledge: acción y su condición — nada despega sin filtro

El `@` del SAMU es el hilo de continuidad en todos los iconos de supervisión:
zipper, headcat, doorman-mobile. La firma del auditor.
