1.0 DEFINICIÓN Y POSICIÓN

====================================================================
1.1 Definición técnica

{doorman-mobile} EXT es la extensión soberana del módulo WORK {doorman-mobile} para operación en contexto de mercado chino: el protocolo específico que permite al guardián de custodia móvil recibir solicitudes de actores chinos via Concierge-Service-Center, operar con UNICODE programable como interfaz nativa, y usar PUENTE-FÁCIL como canal soberano de transferencia entre sistemas de custodia de diferentes jurisdicciones verificadas.

1.2 Relación con BIBLIO-SOURCES(WORK_DOORMAN-MOBILE)

Este documento no reemplaza ni modifica {doorman-mobile} canónico — lo extiende. Todo lo documentado en BIBLIO-SOURCES(WORK_DOORMAN-MOBILE) permanece vigente soberanamente. Esta extensión añade tres capas específicas:

CAPA 1: protocolo de recepción de actores chinos via CSC con UNICODE activo.
CAPA 2: PUENTE-FÁCIL como canal soberano de transferencia entre jurisdicciones de custodia.
CAPA 3: gestión soberana de dispositivos del ecosistema chino con sus características específicas verificadas.

La extensión opera sobre el mismo [foundation] de {doorman-mobile}: soberanía del acceso, verificación de identidad, trazabilidad completa en HL FABRIC, (shield seat) activo permanente.

1.3 Por qué la extensión es necesaria soberanamente

El protocolo canónico de {doorman-mobile} fue diseñado sobre un modelo de dispositivo y jurisdicción. El mercado chino introduce tres variables que el protocolo base no cubre completamente:

Variable 1 — Ecosistema de dispositivos: el mercado chino opera con ecosistemas de hardware soberanamente diferentes (sistemas operativos propios, aplicaciones de pago propias, infraestructura de red soberana). El dispositivo de custodia chino tiene características que {doorman-mobile} base necesita reconocer específicamente.

Variable 2 — Jurisdicción de custodia: los sistemas de custodia chinos operan bajo regulación y control soberano chino, requieren protocolos de transferencia específicos y necesitan canales que respeten ambas jurisdicciones sin comprometer ninguna.

Variable 3 — Interfaz lingüística: los actores chinos operan naturalmente en chino, requieren UNICODE programable como interfaz nativa, y necesitan que el sistema de custodia comprenda matices culturales y contextuales específicos del mercado chino.

2.0 CAPA 1: PROTOCOLO DE RECEPCIÓN CHINA

====================================================================
2.1 Integración con Concierge-Service-Center

El CSC (Concierge-Service-Center) actúa como punto de entrada soberano para actores chinos:

**Flujo de recepción:**
1. Actor chino llega al CSC con solicitud en chino natural
2. CSC activa UNICODE-CHINA para procesamiento semántico
3. CSC verifica identidad según protocolos chinos
4. CSC transfiere solicitud a {doorman-mobile} EXT con contexto completo

**Contexto transferido:**
- Identidad verificada del actor chino
- Intención semántica procesada
- Nivel de acceso determinado
- Dispositivo de origen identificado
- Jurisdicción de operación establecida

**Coordinación UNICODE:**
- UNICODE-CHINA procesa entrada china
- Traducción a tokens LACHO verificables
- Mantenimiento de matices culturales
- Preservación de precisión semántica

2.2 Protocolo de verificación china

La verificación de identidad china incluye:

**Verificación estándar {doorman-mobile}:**
- Identidad soberana LACHO

- (shield seat) activo permanente

**Verificación extendida china:**
- Identidad digital china (身份证数字版)

**Doble verificación soberana:**
- Validación cruzada entre sistemas
- Respeto de ambas jurisdicciones

- Cumplimiento de requisitos chinos

2.3 Gestión de dispositivos chinos

**Reconocimiento de ecosistemas:**

3.0 CAPA 2: PUENTE-FÁCIL COMO CANAL SOBERANO

====================================================================
3.1 Definición de PUENTE-FÁCIL

PUENTE-FÁCIL es el canal económico soberano diseñado específicamente para transferencias de alta frecuencia y baja complejidad entre sistemas de diferentes jurisdicciones. En el contexto chino, PUENTE-FÁCIL opera como el puente ideal para transferencias de custodia entre sistemas LACHO y sistemas chinos.

3.2 Protocolo de transferencia soberana

**Transferencia desde sistema LACHO:**
1. {doorman-mobile} EXT recibe solicitud de transferencia
2. Verificación de permisos y límites soberanos
3. Activación de PUENTE-FÁCIL con contexto chino
4. Transferencia coordinada a sistema chino
5. Confirmación de recepción y registro en HL FABRIC

**Transferencia desde sistema chino:**
1. Sistema chino inicia transferencia via PUENTE-FÁCIL
2. PUENTE-FÁCIL verifica compatibilidad jurisdiccional
3. {doorman-mobile} EXT recibe notificación de llegada
4. Verificación de destino y activación de custodia
5. Confirmación a sistema chino y registro soberano

**Coordinación bilateral:**
- Sincronización de estados entre sistemas
- Manejo de diferencias temporales
- Resolución de conflictos de jurisdicción
- Mantenimiento de integridad de transferencias

3.3 Gestión de jurisdicciones

**Jurisdicción LACHO:**
- Mantenimiento de soberanía sobre activos
- Control de acceso y permisos
- Verificación de identidad LACHO
- Cumplimiento de protocolos LACHO

**Jurisdicción china:**
- Respeto de regulación local
- Cumplimiento de requisitos chinos
- Coordinación con autoridades financieras
- Mantenimiento de compatibilidad sistémica

**Puente soberano:**
- PUENTE-FÁCIL como intermediario neutral
- No compromiso de ninguna soberanía
- Facilitación de transferencias seguras
- Registro completo de operaciones
