## 1.0 ARQUITECTURA OBJETIVO — UNIKERNEL SOBERANO

Unikernel: modelo + drivers + LACHO compilados en un solo binario.
Sin escritorio. Sin servicios innecesarios. Sin bash como init.
El binario es el OS.

Especificaciones objetivo:
  RAM base OS:     ~150MB
  RAM disponible:  resto para modelo cuantizado GGUF
  Hardware actual: i5-4440 · 12GB RAM · Debian/Mint base
  Motor inferencia: llama.cpp con cuantización GGUF
  Lenguaje bridge: Rust (seguridad + velocidad de tensores)

Base recomendada: Debian Minimal / Netinst
  Estabilidad kernel · apt para dependencias numéricas
  Eliminar: GUI · impresión · servicios innecesarios
  Conservar: kernel · red · disco · apt · Python3

## 2.0 TRES ARQUITECTURAS — DE MENOR A MAYOR COMPLEJIDAD

2.1 MICROKERNEL (CAPA C)
  Kernel gestiona solo CPU y RAM.
  LACHO opera en User Space con privilegios altos.
  Lenguaje: Rust
  Estado: CAPA C — PC nueva

2.2 UNIKERNEL (CAPA D objetivo)
  Modelo + drivers + LACHO = un solo binario.
  Se instala directamente en SSD.
  Al arrancar: procesador 100% dedicado al modelo.
  Latencia mínima posible — equivalente a ASIC especializado.
  Estado: CAPA D — DIRIME LINUX completo

2.3 LANGUAGE MODEL OS — LlamaOS (arquitectura conceptual)
  Context Window → RAM (lo que el sistema está pensando)
  Weights (pesos) → CPU/GPU (conocimiento fijo)
  Tools           → drivers directos a disco y red
  Interrupciones  → input soberano → respuesta LACHO
  Diferencia con Gemini: LACHO agrega gramática verificable
  entre el input y el modelo — sin alucinaciones de comandos.

## 3.0 VENTAJA SOBERANA SOBRE PROPUESTA GEMINI

Gemini propone: hardware → modelo → output
DIRIME implementa: humano → LACHO → SAMU → ledger → modelo → output

La diferencia es trazabilidad soberana:
  - Gemini no tiene gramática de verificación intermedia
  - Sin LACHO el modelo puede alucinar comandos del sistema
  - El ledger inmutable no existe en ninguna propuesta Gemini
  - STACKING es el Semantic File System que Gemini propone
    como concepto futuro — DIRIME ya lo tiene operativo en IMV
